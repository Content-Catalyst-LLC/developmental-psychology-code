# Accessibility Framework

## Developmental access variables

This workflow treats access as developmental infrastructure.

| Domain | Developmental relevance |
|---|---|
| Communication access | Supports agency, consent, learning, relationship, and self-advocacy |
| Physical access | Supports mobility, independence, participation, and public life |
| Sensory access | Reduces avoidable distress and enables regulation |
| Educational access | Supports learning, skill development, and belonging |
| Assistive technology | Expands communication, mobility, cognition, and autonomy |
| Service access | Shapes early support, continuity, and family burden |
| Institutional flexibility | Determines whether differences are accommodated or punished |

## Interpretation

The model does not ask whether disabled or neurodivergent people are inherently limited. It asks how developmental outcomes change when support rises, access improves, and barriers fall.
