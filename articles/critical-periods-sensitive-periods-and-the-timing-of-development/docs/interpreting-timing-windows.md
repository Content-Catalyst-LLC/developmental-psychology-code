# Interpreting Timing Windows

## Critical periods

A critical-period model assumes a relatively narrow window in which a given input is necessary or unusually decisive. In code, this is represented with a binary indicator.

## Sensitive periods

A sensitive-period model assumes heightened responsiveness around a developmental peak. In code, this is represented with a smooth timing weight, usually a bell-shaped curve.

## Multi-window development

Some developmental systems may show more than one period of heightened sensitivity. This repository includes both an early sensitive window and an adolescent sensitive window to reflect the idea that developmental timing is distributed, not confined to the earliest years.

## Timing without determinism

The examples include a later-intervention/recovery component to show that timing effects can be strong without implying total closure.
